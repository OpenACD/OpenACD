Description

A tcp listener for agent tcp clients using json as the encoding protocol. This is useful for desktop applications, or if a web poll is not desired. The Json api is the same as the web api, allowing it to keep pace (aside from obvious protocol mismatches, like path fetching).

Json, after being turned into a binary by mochijson2:encode/1, is wrapped in a netstring:

	[len] ":" [json] ","

"len" being the the stringification of the length of the protobuf. So, a netstring of the binary <<"hello">> would look like <<"5:hello,">>. For a long discussion, http://cr.yp.to/proto/netstrings.txt.

When a client first connects, they will get a message stating the agent is in the state PRELOGIN. That is the signal to start the login in ernest. After that an agent should:
  1. Verify it's version
  2. Request a salt
  3. Request to login

The version verification will reply an error if the major version doesn't match. If only the minor version part doesn't match, success is returned, but error_message is populated with a message stating as much. A perfect match is, well, perfect.

The request for a salt gets a nonce to use for salting the password, as well as the E and N parts of a public key to rsa encrypt the password.

Using the information from the get salt step, encrypt the password. Then send a login request. If successful, the server will reply as such, and an event stating which state the agent is in should also arrive.

Encrypting the password is skipped if the connection is over SSL.

start_link/1

start_link(Opts::start_opts()) -> {ok, pid()} | ignore | {error, any()}

Start the listener linked to parent process. The Options (and defaults) are:

port :: pos_integer()
7331 Port to listen for incoming connections.
radix :: pos_integer()
10 Represent netstring lengths in the given numberical base. Minimum is 2, maximum is 36.
socket_type
tcp If tcp, passwords must be encrypted by rsa using the requested salt. If ssl_upgrade, the socket must be upgraded to ssl after the connection is established. If ssl, the initial connection must be over ssl. In both ssl options, the password is not encrypted (other than what ssl does).
zip_json :: boolean()
false If present, true, or zip, the json binary is put through zlib:zip/1 before being wrapped in a netstring. If gzip, the binary is put through zlib:gzip/1 instead. If missing or false, the json is not compressed. The client is assumed to be using the same compression scheme.
poolsize :: pos_integer
5 How many acceptor processes to start for the listening socket.
